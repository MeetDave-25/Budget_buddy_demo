## üìã How to Request Permission

### For Using the Project
If you want to use this project for any purpose:

1. **Open an Issue** on this repository with the title: "Permission Request: [Your Use Case]"
2. **Include the following information:**
   - Your name and contact information
   - Detailed description of how you plan to use the project
   - Whether your use is commercial or non-commercial
   - Any modifications you plan to make
   - Timeline for your intended use

3. **Wait for approval** from the project maintainer (Meet G. Dave)
4. Once approved, you will receive written permission to proceed

### For Contributing to the Project
If you want to contribute improvements or fixes:

1. **First, request permission** by opening an issue with the title: "Contribution Permission Request"
2. **Describe your proposed contribution:**
   - What feature or fix you want to implement
   - Why it would benefit the project
   - Your estimated timeline

3. **Wait for approval** before proceeding
4. Once approved, follow the contribution workflow below

## üîÑ Contribution Workflow (After Permission is Granted)

### 1. Fork the Repository
- Only fork after receiving explicit permission
- Forks are only permitted for the purpose of contributing back to this project
- Do NOT use your fork for any other purpose without separate permission

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Follow the existing code style and conventions
- Write clear, descriptive commit messages
- Test your changes thoroughly
- Ensure no TypeScript errors or warnings

### 4. Commit Your Changes
```bash
git add .
git commit -m "feat: add your feature description"
```

Use conventional commit messages:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for formatting changes
- `refactor:` for code refactoring
- `test:` for adding tests
- `chore:` for maintenance tasks

### 5. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 6. Create a Pull Request
- Provide a clear title and description
- Reference any related issues
- Explain what changes you made and why
- Include screenshots for UI changes
- Wait for review and approval

## üìù Code Standards

### TypeScript
- Use proper TypeScript types (no `any` types)
- Import types from `src/types/index.ts`
- Follow existing patterns for component props

### React Components
- Use functional components with hooks
- Follow the existing component structure
- Use proper prop types and interfaces

### Styling
- Use Tailwind CSS classes
- Follow the existing color scheme and design patterns
- Ensure responsive design works on mobile and desktop

### File Organization
- Place new components in `src/components/`
- Place new types in `src/types/index.ts`
- Follow the existing folder structure
